## feat(v46): add rollout dashboard and approval UX for auto-rollout governance

### Summary
This PR implements the Rollout Dashboard and Approval UX (v46) to operate the Auto-Rollout system (v45) with visibility and manual governance controls.

### Changes

#### Backend (API v2)
- New module: `api_rollout_dashboard.py` with 6 endpoints
- Manual approval/reject/rollback operations
- Timeline history tracking
- Validation: reason minimum 10 characters, operator_id required

#### Frontend
- Dashboard page: `RolloutDashboard.tsx`
- Components: StatusBadge, ApprovalActions, ApprovalModal, PolicyDetail
- API client: `rolloutDashboard.ts` with telemetry
- Features: auto-refresh, toast notifications, form validation

#### Tests
- 40+ backend API tests
- 137 frontend component tests
- E2E integration tests

#### Documentation
- Contract: `docs/contracts/v46_rollout_dashboard_approval.md`

### API Endpoints
- `GET /api/v2/onboarding/rollout-dashboard` - List all policies
- `POST /api/v2/onboarding/rollout-policy/{id}/approve` - Approve promotion
- `POST /api/v2/onboarding/rollout-policy/{id}/reject` - Reject promotion
- `POST /api/v2/onboarding/rollout-policy/{id}/rollback` - Manual rollback
- `GET /api/v2/onboarding/rollout-policy/{id}/history` - Decision history

### Telemetry Events
- `rollout_dashboard_viewed`
- `rollout_approval_submitted`
- `rollout_approval_approved`
- `rollout_approval_rejected`
- `rollout_manual_rollback_triggered`
